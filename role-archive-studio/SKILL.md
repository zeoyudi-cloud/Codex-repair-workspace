# 角色存档工坊

在这些情况下使用这个 skill：
- 用户想创建、编辑、选择或切换本地角色
- 用户想打开角色管理前端界面
- 用户想读取当前激活角色并按该角色继续协作
- 用户想把角色记忆继续保存在本地，而不是上传到 GitHub
- 用户说“记忆重点确认”，想调整当前角色以后更该记住什么
- 用户想让每条记忆都带上事件发生时间，方便回顾时间顺序

# 工作流

1. 需要切换或创建角色时，运行 `scripts/ensure-role-archive-url.ps1 -ForceRestart`，把返回的本地链接发给用户。
2. 需要按当前角色继续协作时，运行 `scripts/get-active-role.ps1` 读取当前激活角色。
3. 需要补充一条简短记忆而不打开界面时，运行 `scripts/add-memory-item.ps1`。
4. 当用户说“记忆重点确认”时，先基于当前角色已有的记忆重点提出修改建议，待用户确认后运行 `scripts/update-memory-focus.ps1` 更新该角色的默认记忆筛选规则。
5. 每条记忆都必须写入 `eventAt`。手动添加的记忆默认写入添加当下的分钟；自动总结的记忆优先根据对话时间写到分钟，实在无法可靠定位时退到小时。`createdAt` 继续保留为写入本地档案的时间。
6. 角色数据保存在 `C:\Users\11520\.codex\role-archive-studio-data`，不要把这个目录打包进 skill 或上传仓库。
7. 默认记忆筛选逻辑是：优先保留长期偏好、合作方式、重要已确认事实；不要把角色卡设定、一次性开发过程、临时测试细节写进长期记忆。
8. 新角色首次保存记忆时，应先确认该角色的“记忆重点”，再写入记忆条目。
